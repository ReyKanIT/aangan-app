KEY FUNCTIONS (4 total)
================================================================================

1. check_storage_limit(user_id, file_size_bytes) → BOOLEAN
   └─ Purpose: Check if user can upload file
   └─ Logic: Personal storage first, then pool storage
   └─ Returns: TRUE if space available, FALSE if quota exceeded
   └─ Usage: Called before every file upload
   └─ Security: DEFINER function (can access storage data)

2. increment_early_adopter_count() → BOOLEAN
   └─ Purpose: Increment early adopter counter (max 10K)
   └─ Logic: Reads config, increments if under limit
   └─ Returns: TRUE if incremented, FALSE if limit reached
   └─ Usage: Called on user signup
   └─ Security: DEFINER function (can update config)

3. create_user_storage_on_signup() → TRIGGER
   └─ Purpose: Auto-create user_storage record on signup
   └─ Logic: Generates unique referral code (WHILE loop for uniqueness)
   └─ Execution: AFTER INSERT on public.users
   └─ Defaults: 10GB base, 0 referral bonus, tier='base'

4. update_updated_at_column() → TRIGGER
   └─ Purpose: Auto-update timestamp on any write
   └─ Used by: 5 tables (user_storage, family_storage_pools, app_config, etc.)
   └─ Execution: BEFORE UPDATE on each table

================================================================================
DATA FLOW EXAMPLES
================================================================================

1. USER SIGNUP → AUTO STORAGE CREATED
   User signup → auth.users trigger → handle_new_user() 
   → public.users insert → create_user_storage_on_signup() trigger 
   → user_storage record created with:
      - user_id: new user
      - base_storage_gb: 10
      - referral_code: unique (8-char)
      - storage_tier: 'base'

2. REFERRAL VERIFICATION
   User A invites User B (referral_code="abc123")
   → User B signs up with code
   → referrals row created (status='pending', referred_at=now)
   → User B active for 7+ days
   → System marks verified (status='verified', verified_at=now)
   → User A storage updated: +2GB referral bonus, verified_referral_count++
   → User A tier may change: 'base' → 'bronze' if qualifies

3. EVENT BUNDLE PURCHASE
   Event creator creates event
   → Selects bundle type (e.g., Mangal @ ₹1,499)
   → Initiates Razorpay payment
   → Payment processed → razorpay_payment_id stored
   → event_bundles row created:
      - bundle_type: 'mangal'
      - storage_gb: 200
      - gallery_expires_at: now + 1 year
      - status: 'active'
   → Guests upload photos (separate bucket: event-bundles/{event_id}/)
   → Gallery expires → download warning → archive → delete

4. PHOTO UPLOAD WITH STORAGE LIMIT CHECK
   User uploads photo to event
   → check_storage_limit(user_id, file_size) called
   → Checks personal storage: if available → upload allowed
   → If personal full → checks pool storage (if member)
   → If both full → upload rejected
   → event_photos row created (status='pending')
   → Event creator approves (status='approved')
   → Photo visible to invited guests only
