# Aangan v0.2 — Security Release

**Date:** March 31, 2026
**Version:** 0.2.0 (Build 2)

## What's New in v0.2

### 1. Server-Side Rate Limiting
- **Supabase Edge Function** (`/functions/v1/rate-limit`) enforces limits at the server level
- Protects: OTP sends (5/10min), OTP verify (5/5min), login attempts (10/15min), post creation (20/hr), reports (10/hr)
- Automatic blocking with configurable cooldown periods
- **Database table** (`rate_limits`) tracks attempts server-side — no more client-only enforcement
- Cleanup function removes expired entries automatically

### 2. Admin Dashboard (Web)
- **New `/admin` route** in the Next.js web app with 5 pages:
  - **Dashboard** — user counts, post/event stats, pending reports
  - **User Management** — search, deactivate, toggle admin status
  - **Content Reports** — review pending reports, resolve/dismiss with notes
  - **Audit Log** — searchable history of all security-relevant actions
  - **App Settings** — feature flags, maintenance mode, version control
- Protected by `is_app_admin` flag + middleware auth check

### 3. Content Reporting & Moderation
- **New `content_reports` table** with full lifecycle: pending → reviewing → resolved/dismissed
- **React Native screens**:
  - `ReportContentScreen` — report posts, photos, events, or users
  - 6 report reasons: inappropriate, spam, harassment, fake account, privacy violation, other
- **Report store** (`reportStore.ts`) for state management
- Admin moderation queue in the web dashboard
- Unique constraint prevents duplicate reports

### 4. Audit Logging
- **New `audit_logs` table** captures all security-relevant events
- Events tracked: login, logout, profile update, post/event create/delete, family add/remove, photo moderation, report resolution, admin actions, account deactivation
- **Edge Function** (`/functions/v1/audit-log`) for server-side insertion
- **SECURITY DEFINER** function `log_audit_event()` for database-level logging
- Admin-only visibility via RLS
- Captures IP address and user agent

### 5. User Blocking
- **New `user_blocks` table** with bidirectional enforcement
- **Block store** (`blockStore.ts`) — block/unblock users, check block status
- Self-block prevention via CHECK constraint
- Admin can view all blocks

### 6. App Settings / Feature Flags
- **New `app_settings` table** — key-value store for runtime configuration
- Default settings: maintenance_mode, min_app_version, max_otp_attempts, registration_open
- Admin-managed via web dashboard
- All authenticated users can read (for feature flag checks)

### 7. Admin RLS Policies
- **40+ new RLS policies** giving `is_app_admin` users full access across all 12 existing tables
- Admins can SELECT, UPDATE, DELETE on: users, posts, events, event_rsvps, family_members, notifications, event_photos, event_checkins, event_confirmations, physical_cards, audience_groups, post_audience

### 8. Customer Support Screens (React Native)
- **HelpScreen** — 15 FAQ items in 5 sections, expandable accordions, Hindi/English
- **FeedbackScreen** — category picker, message input, mailto submission
- **Terms of Service** — comprehensive ToS for Indian family social network
- **Privacy Policy** — GDPR-inspired privacy policy with Hindi support
- All accessible from Settings screen

### 9. Enhanced Security Utils
- **Content sanitization**: HTML escape, suspicious pattern detection, XSS prevention
- **Phone masking**: `maskPhoneNumber()` for display security
- **Blocked file extensions**: prevents dangerous file uploads (exe, bat, sh, etc.)
- **Display name sanitization**: strips injection characters
- **Deep link validation**: only allows https, mailto, tel schemes
- **Session timeout detection**: `isSessionExpired()` helper

### 10. Analytics & Crash Reporting (Ready)
- **`analytics.ts`** wrapper with placeholder methods
- Ready for Sentry (`@sentry/react-native`) and PostHog integration
- Methods: `init`, `trackScreen`, `trackEvent`, `trackError`, `setUser`, `reset`, `breadcrumb`

## Deployment Steps

### 1. Run Database Migration
```sql
-- In Supabase SQL Editor, run:
-- supabase_migration_v0.2_security.sql
```

### 2. Deploy Edge Functions
```bash
supabase functions deploy rate-limit
supabase functions deploy audit-log
```

### 3. Set First Admin
```sql
-- In Supabase SQL Editor:
UPDATE public.users SET is_app_admin = TRUE WHERE phone_number = '+91XXXXXXXXXX';
```

### 4. Verify
- Visit `/admin` on the web app — should show dashboard
- Try reporting content from the mobile app
- Check audit logs populate after admin actions
